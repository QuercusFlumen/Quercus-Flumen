# Experimentación
## Tea — Ensayos, observaciones y validación práctica

La carpeta `experimentacion/` reúne la documentación técnica derivada de los ensayos realizados con la materia prima del proyecto **Quercus Flumen — Tea**.

Su función es registrar de forma ordenada la experiencia real de prueba, comparación y descubrimiento que permite pasar de una intuición de producto a un sistema con comportamiento observable y reproducible.

Aquí se documentan:

- hipótesis de trabajo
- configuraciones de ensayo
- resultados comparativos
- observaciones visuales
- parámetros físico-químicos
- conclusiones provisionales
- implicaciones para el desarrollo del producto

Esta carpeta nace porque la evolución del proyecto ha superado la fase de prueba informal.

Los ensayos realizados muestran ya patrones suficientemente claros como para requerir una documentación propia.

En particular, la experimentación ha permitido detectar:

- una ventana de extracción especialmente eficaz
- un comportamiento comparativo reproducible
- la obtención de un concentrado líquido estable
- una diferencia clara entre simple infusión y extracto concentrado

Desde el punto de vista del proyecto, esta carpeta cumple tres funciones.

## 1. Registrar el proceso real de descubrimiento

No solo interesa el resultado final.

Interesa también conservar cómo se llegó a él:

- qué se probó
- qué funcionó
- qué empeoró el resultado
- qué patrones comenzaron a repetirse

## 2. Fijar conocimiento técnico interno

A medida que los ensayos muestran coherencia, el conocimiento deja de ser intuición y empieza a convertirse en protocolo.

La carpeta permite fijar ese conocimiento con lenguaje técnico y útil para el desarrollo posterior.

## 3. Preparar la futura traducción pública

Parte de lo aprendido aquí podrá trasladarse más adelante a la documentación pública del proyecto.

Sin embargo, esta carpeta mantiene una función interna y experimental, por lo que puede contener formulaciones más directas, comparativas y orientadas a la validación del sistema.
